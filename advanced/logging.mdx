# Logging

The ROS2 Installer CLI provides comprehensive logging capabilities to help you monitor installation progress and troubleshoot issues.

## Log Configuration

Logging is configured in your `config_cli.yaml` file:

```yaml
logging:
  level: 'INFO'              # DEBUG, INFO, WARNING, ERROR, CRITICAL
  file: '/tmp/ros2_installation.log'
  console: true              # Display logs in terminal
```

## Log Levels

The installer supports standard Python logging levels:

### DEBUG

Most verbose level, includes all execution details:
- Command execution details
- Configuration loading steps
- System checks
- Package management operations

```bash
sudo python3 ros2_installer_cli.py --log-level DEBUG
```

**Use case:** Detailed troubleshooting and development

### INFO (Default)

Standard operational information:
- Installation progress
- Package installation status
- System validation results
- Major operation completions

```bash
sudo python3 ros2_installer_cli.py --log-level INFO
```

**Use case:** Normal operation monitoring

### WARNING

Potential issues that don't prevent installation:
- Optional package failures
- System info gathering errors
- Non-critical configuration issues

```bash
sudo python3 ros2_installer_cli.py --log-level WARNING
```

**Use case:** Production installations where you only want to see problems

### ERROR

Serious problems that may cause installation failure:
- Package installation failures
- Validation errors
- Repository setup issues

```bash
sudo python3 ros2_installer_cli.py --log-level ERROR
```

**Use case:** Minimal logging for automated systems

### CRITICAL

Severe errors causing installation termination:
- System validation failures
- Configuration errors
- Unrecoverable installation errors

```bash
sudo python3 ros2_installer_cli.py --log-level CRITICAL
```

**Use case:** Only show fatal errors

## Log File Locations

### Default Location

**Path:** `/tmp/ros2_installation.log`

This is the primary log file location (configurable in `config_cli.yaml`).

```bash
# View real-time logs
tail -f /tmp/ros2_installation.log

# Search for errors
grep -i error /tmp/ros2_installation.log

# View last 100 lines
tail -n 100 /tmp/ros2_installation.log
```

### Fallback Location

**Path:** `~/ros2_installation_<session_id>.log`

If the default location is not writable (permission denied), logs are written to your home directory with a unique session ID.

**Implementation** (source:ros2_installer_cli.py:186-203):

```python
try:
    file_handler = logging.FileHandler(log_file)
    # Configure handler...
except PermissionError:
    # Fallback to user home directory
    log_file = os.path.expanduser(f"~/ros2_installation_{self.session_id}.log")
    file_handler = logging.FileHandler(log_file)
```

### Find Your Log File

```bash
# Check default location
ls -lh /tmp/ros2_installation.log

# Find fallback logs in home directory
ls -lh ~/ros2_installation_*.log

# Find most recent log
ls -lt ~/ros2_installation_*.log | head -1
```

## Log Format

### File Log Format

Logs written to file include session ID for tracking:

```
2026-03-04 10:15:23,456 - ros2_cli_installer - INFO - [a1b2c3d4] - 🚀 ROS2 CLI Installer v2.1.0
2026-03-04 10:15:23,457 - ros2_cli_installer - INFO - [a1b2c3d4] - Session ID: a1b2c3d4
2026-03-04 10:15:23,458 - ros2_cli_installer - INFO - [a1b2c3d4] - System: Linux 5.15.0
```

**Format:**
```
%(asctime)s - %(name)s - %(levelname)s - [%(session_id)s] - %(message)s
```

**Source:** ros2_installer_cli.py:190-192

### Console Log Format

Simplified format for terminal output:

```
2026-03-04 10:15:23,456 - INFO - 🚀 ROS2 CLI Installer v2.1.0
2026-03-04 10:15:23,457 - INFO - Session ID: a1b2c3d4
2026-03-04 10:15:23,458 - INFO - System: Linux 5.15.0
```

**Format:**
```
%(asctime)s - %(levelname)s - %(message)s
```

**Source:** ros2_installer_cli.py:208-210

## Session ID

Each installation run gets a unique 8-character session ID for tracking:

```python
self.session_id = str(uuid.uuid4())[:8]  # e.g., "a1b2c3d4"
```

**Use cases:**
- Track multiple installation attempts
- Correlate logs across different components
- Identify backup files created during installation
- Debug concurrent installations

**Source:** ros2_installer_cli.py:72

    """Setup comprehensive logging system."""
    logger = logging.getLogger('ros2_cli_installer')
    logger.setLevel(getattr(logging, self.config['logging'].get('level', 'INFO')))
    
    # Clear existing handlers
    logger.handlers.clear()
    
    # File handler with session ID
    # Console handler (optional)
    # Session filter for tracking
    
    return logger

## Disabling Console Output

To disable console logging (file only):

```yaml
logging:
  level: 'INFO'
  file: '/tmp/ros2_installation.log'
  console: false  # Disable terminal output
```

**Use case:** Silent installations, automated scripts, CI/CD pipelines

```bash
sudo python3 ros2_installer_cli.py --silent
```

## Command Execution Logging

All shell commands are logged with execution time:

```python
def _run_command(self, command: str, description: str = "", 
                  timeout: int = None, retries: int = None):
    self.logger.debug(f"Executing: {command}")
    # ... execution ...
    duration = time.time() - start_time
    self.logger.info(f"✅ {description} (took {duration:.2f}s)")
```

**Example output:**
```
2026-03-04 10:15:30 - DEBUG - Executing: sudo apt-get update
2026-03-04 10:15:35 - INFO - ✅ Updating package index (took 5.23s)
```

**Source:** ros2_installer_cli.py:438-476

## Retry Logic Logging

Failed commands are logged with attempt numbers:

```
2026-03-04 10:20:15 - WARNING - ⏰ Command timed out (attempt 1/3): sudo apt-get install...
2026-03-04 10:25:20 - WARNING - ⏰ Command timed out (attempt 2/3): sudo apt-get install...
2026-03-04 10:30:25 - INFO - ✅ Installing desktop-full packages (took 610.45s)
```

## Error Logging

Errors include full context for troubleshooting:

```python
except subprocess.CalledProcessError as e:
    self.logger.warning(f"❌ Command failed (attempt {attempt + 1}/{retries}): {command}")
    self.logger.warning(f"Exit code: {e.returncode}")
    if attempt == retries - 1:
        raise
```

- GPG signature checks

## Log Analysis

### Extract session summary

```bash
grep "Session ID" /tmp/ros2_installation.log
grep "completed successfully" /tmp/ros2_installation.log
```

### Find all errors

```bash
grep -i "error\|failed\|❌" /tmp/ros2_installation.log
```

### Find all warnings

```bash
grep -i "warning\|⚠️" /tmp/ros2_installation.log
```

### Track specific session

```bash
grep "\[a1b2c3d4\]" /tmp/ros2_installation.log
```

### Performance analysis

```bash
grep "took.*s)" /tmp/ros2_installation.log
```

## Integration with Configuration

Override log level via command line:

```bash
sudo python3 ros2_installer_cli.py --log-level DEBUG
```

This overrides the configuration file setting:

```python
if args.log_level:
    installer.config['logging']['level'] = args.log_level
    installer.logger.setLevel(getattr(logging, args.log_level))
```

## Best Practices

1. **Always check logs** after installation failures
2. **Use DEBUG level** for troubleshooting specific issues
3. **Keep session IDs** for tracking multiple installation attempts
4. **Archive logs** for long-term debugging and auditing
5. **Use grep/awk** for log analysis and pattern matching
6. **Monitor disk space** if logging extensively
